= ONTAPアラートの前提条件
:hardbreaks:
:allow-uri-read: 
:icons: font
:imagesdir: ./media/get-started/


[role="lead"]
運用環境の準備状況を確認して、 ONTAPアラートを開始します。

ONTAPアラートを表示するには、次の前提条件が必要です。

* ONTAP 9.10 以降。オンプレミス システムは、 NetAppコンソール システム ページで検出される必要があります。
+
** クラスタ管理者のONTAP権限
+
ライセンスをNetAppコンソールのデジタル ウォレットに入力する必要はありません。



* NetAppコンソールの場合:
+
** NetAppオンプレミス NAS または SAN ストレージを備えたNetAppコンソール システム
** オンプレミス リージョンのコンソール エージェントをNetAppコンソールで設定する必要があります。参照 https://docs.netapp.com/us-en/cloud-manager-setup-admin/concept-connectors.html["コンソールエージェントを構成するためのNetAppコンソールドキュメント"^]。
+
クラスターがコンソール エージェントから削除された場合、そのクラスターに関連するアラートは 90 日間残ります。また、コンソール エージェントを変更した場合、アラートは元のクラスターとエージェントに関連付けられたまま 90 日間残ります。元のエージェントに戻ると、それらのアラートが再び表示されます。

** オンプレミスのONTAPクラスターに接続するアクティブなコンソール エージェントが少なくとも 1 つあるNetAppコンソール組織。すべてのソースとシステムは同じNetAppコンソール組織内にある必要があります。
** リソースを検出するための組織管理者権限を持つ、その組織内のNetAppコンソール ユーザー
** https://docs.netapp.com/us-en/cloud-manager-setup-admin/reference-checklist-cm.html["標準的なNetAppコンソールの要件"^]
